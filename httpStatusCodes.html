Some of the most common status codes samples on web are:

403 – Forbidden which means, the server can accept the request but refused to take/make an action

404 – The request to the server cannot fulfill because the resource cannot be found or is unavailable.

500 – When the server cannot be process the request as of the moment.
  • HTTP Status Codes
  • Status codes are issued by a server in response to a client's request made to the server. IETF RFC gives the codes used in some common applications of the HTTP. The standard classes of codes is listed in a numerical identity and are described in a human-readable phrases or otherwise codes based on the standards in RFC 7231. These codes are separated into five classes.

    According to RFC 7231 the codes being described are:

    1XX – Informational (The request was received, continuing process)
    2XX – Successful (The request was successfully received, understood, and accepted)
    3XX – Redirection (Further action needs to be taken in order to complete the request)
    4XX - Client Error (The request contains bad syntax or cannot be fulfilled)
    5XX - Server Error (The server failed to fulfill an apparently valid request)

    Conclusion: These codes are given and standardized to be a helping tool to troubleshoot the issues or describe the current behavior of your web application(s). There are many common solutions that can be found on the web resources and guide you to familiarize these status codes and save more time in the development.
